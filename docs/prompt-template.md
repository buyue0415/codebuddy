# 新增功能开发提示词模板

直接复制使用，替换 `[ ]` 中的内容即可。

---

## 模板 1：全新功能

> 用于创建之前不存在的新模块/新功能。

```markdown
## 需求：[一句话说明功能]

## 实现要求
[具体的技术细节、输入输出、行为描述]

## 请确保
1. 创建对应的 spec 文档（docs/specs/）
2. 新增完整的测试用例（tests/）
3. 更新 .rules 文件（如有架构性变化）
```

## 模板 2：修改已有功能

> 用于修改、重构、优化已有功能。

```markdown
## 需求：修改 [模块名]

## 变更点
- 原行为：[旧逻辑]
- 新行为：[新逻辑]

## 请确保三者同步
- docs/specs/XX-xxx.md 中对应章节同步更新
- 已有测试用例更新 + 新增边界测试
- 如有架构变更，更新 .rules
```

## 模板 3：极简版

> 懒得写细节时用，但我可能反过来问你。

```markdown
实现 [功能]，同步更新 spec、test、rules
```

## 模板 4：修 Bug

> 修复 Bug 时建议带上回归测试，防止再次出现。

```markdown
## Bug：[描述问题]

## 根因分析
[导致 Bug 的原因]

## 修复要求
[具体修改方式]

## 请确保
1. 新增回归测试覆盖该场景
2. 更新相关 spec（如果文档描述与实际行为不符）
```

---

## 提示词里写什么，AI 就会做什么

| 提示词里说到的 | AI 默认会做 |
|---------------|-----------|
| 只写了功能要求 | ✅ 改代码 |
| 写了 + 更新 spec | ✅ 改代码 + ✅ 更新 docs |
| 写了 + 更新 test | ✅ 改代码 + ✅ 写测试 |
| 写了 + 更新 rule | ✅ 改代码 + ✅ 更新 .rules |
| 写了三者同步 | ✅ 全部做 |
