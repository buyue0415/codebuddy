## 1. 业务需求说明书

### 1.1 业务背景

投资者需要在不使用真实资金的情况下验证交易策略的有效性。系统提供虚拟账户、基于预测信号的自动交易建议和执行功能。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 虚拟账户 | 初始资金、余额、市值、总盈亏 |
| 模拟持仓 | 虚拟买入/卖出持仓管理 |
| 交易建议 | 基于ML预测生成的买卖建议 |
| 自动执行 | 后台线程自动执行交易建议 |
| 资金曲线 | 账户净值走势图（Chart.js） |
| 交易记录 | 模拟交易历史明细 |
| 日内分时 | 日内分时行情采集和展示 |
| 账户重置 | 一键重置虚拟账户 |

## 3. 业务逻辑

### 3.1 虚拟账户模型

```javascript
// 账户状态
{
  initial_capital: 100000,   // 初始资金
  balance: 85000,            // 现金余额
  market_value: 32000,       // 持仓市值
  total_pnl: 17000,          // 总盈亏
  total_trades: 12,          // 交易总次数
  last_sync: "2026-06-16 17:10:00"
}

// 仓位管理（凯利公式）
function calcKellyPosition(winRate, avgWin, avgLoss) {
    // f* = (p × b - q) / b
    // p = 胜率, q = 1-p, b = 平均盈亏比
    return (winRate * avgWin - (1 - winRate) * Math.abs(avgLoss)) / avgWin
}
```

## 4. 交互流程

```
挂载 → GET /api/v2/paper/account → 账户概览
  → GET /api/v2/paper/positions → 当前持仓
  → GET /api/v2/paper/suggestions → 交易建议
  → GET /api/v2/paper/trades → 交易记录
  → GET /api/v2/paper/performance → 资金曲线数据
```

### 4.1 执行交易
```
点击 [执行交易] → POST /api/v2/paper/execute
  → 后端: 根据建议列表逐条执行买入/卖出
  → 更新: account / positions / trades
  → 前端: 刷新所有纸面交易数据
```

### 4.2 重置账户
```
点击 [重置账户] → 确认弹窗 → POST /api/v2/paper/reset
  → 后端: 清空持仓/交易/建议，重置余额
  → 前端: 重新加载账户数据
```
