## 1. 业务需求说明书

### 1.1 业务背景

投资者需要及时了解自选股相关的新闻资讯，并分析新闻情感对股价的潜在影响。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 新闻浏览 | 按时间线展示股票相关新闻 |
| 情感分析 | 利好/利空/中性情感标签 |
| 重大事件 | 高亮标记重大事件新闻 |
| 股票筛选 | 按股票代码过滤新闻 |
| 日期选择 | 自定义日期范围查询 |
| 情感趋势 | 按日统计情感趋势堆叠图 |
| 新闻采集 | 一键触发后端新闻采集 |

## 3. 业务逻辑

### 3.1 新闻数据

从 `news` 表获取，默认按日期降序排列：

| 字段 | 说明 |
|------|------|
| title | 新闻标题 |
| summary | 摘要 |
| source | 来源（东方财富/证券时报/财联社等） |
| sentiment | 情感: bullish(利好🟢)/bearish(利空🔴)/neutral(中性⚪) |
| is_major | 是否重大事件⭐ |
| date | 日期 YYYY-MM-DD |
| url | 原文链接 |

// 情感标签样式

## 4. 交互流程

```
挂载 → 加载所有新闻 from useDataStore.allNews
  → 默认显示"全部"视图
  → 筛选 → 过滤列表 + 更新趋势图
  → 点击新闻 → 弹窗显示详情
  → 点击原文 → window.open(url)
```

### 4.1 新闻采集
```
点击 [刷新] → POST /api/trigger/news
  → 后端 subprocess → fetch_news.py
  → 采集自选股最新新闻 → 写入 SQLite
  → 完成后前端重载 news 数据
```
