## 1. 业务需求说明书

### 1.1 业务背景

投资者需要验证交易策略的历史表现，通过回测引擎对策略进行 Walk-forward 两阶段搜索验证。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| Walk-forward 回测 | 滚动训练窗口 + 测试窗口 |
| 进度监控 | 实时显示回测进度百分比 |
| 结果分析 | 回测完成后的交易明细和盈亏统计 |
| 历史记录 | 查看过往回测运行结果 |
| 取消操作 | 支持取消正在运行的回测任务 |

## 3. 业务逻辑

### 3.1 Walk-forward 回测策略

```python
def run_backtest(train_window=252, test_window=21, codes=None):
    """Walk-forward 两阶段搜索:
    1. 在训练窗口内寻找最佳参数组合
    2. 在测试窗口内验证策略表现
    3. 滚动窗口向前移动
    """
    for window in sliding_windows(train_window, test_window):
        # Phase 1: 训练阶段
        best_params = grid_search(window.train_data)
        
        # Phase 2: 测试阶段
        results = validate_strategy(window.test_data, best_params)
        save_backtest_trades(results)
        
        update_progress(current_window / total_windows * 100)
```

### 3.2 回测参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| train_window | 252 | 训练窗口（约1年交易日） |
| test_window | 21 | 测试窗口（约1个月） |
| codes | 全自选股 | 回测股票范围 |

### 3.3 回测结果指标

| 指标 | 计算方式 |
|------|----------|
| 总收益率 | 期末净值 / 期初净值 - 1 |
| 年化收益率 | (1 + 总收益率)^(252/总交易日) - 1 |
| 夏普比率 | 超额收益 / 收益标准差 |
| 最大回撤 | max(峰值到谷底的跌幅) |
| 胜率 | 盈利交易次数 / 总交易次数 |

## 4. 交互流程

```
挂载 → 渲染配置面板 + 历史记录列表

启动回测:
  配置参数 → 点击 [启动回测]
  → POST /api/v2/backtest/run {train_window, test_window, codes}
  → 后端 subprocess → backtest_engine.py
  → 前端轮询 GET /api/v2/backtest/status (每2秒)
  → 更新进度条 → 完成后加载结果

停止回测:
  点击 [停止] → POST /api/v2/backtest/stop
  → 设置取消标志 → 回测进程退出

查看结果:
  GET /api/v2/backtest/results/{run_id}
  → 渲染交易明细表 + 统计指标
```
