## 1. 业务背景

### 1.1 现状

当前系统为本地银行股投资管理工具，已具备以下核心能力：

| 已有能力 | 说明 |
|---------|------|
| 日K线数据采集 | 每只自选股200~2000条（默认拉取上限2000条），每日自动更新 |
| 10信号技术分析 | MACD/RSI/Bollinger/KDJ/ADX/OBV等 |
| 自学习预测 (MWU) | 每日收盘后根据实际涨跌自动调权，生成10天预测 |
| ML元学习器 | RandomForest 学习"规则何时可靠"，调整置信度 |
| 真实持仓管理 | 广发对账单导入，持仓/盈亏/分红分析 |
| 新闻/行情/季节性 | 辅助信息聚合 |

### 1.2 核心痛点

| # | 痛点 | 影响 |
|----|------|------|
| 1 | **MWU在线学习冷启动慢** | 新加入自选股权重从1.0开始，需10-20天才能调优。期间预测质量不佳，用户无法信任 |
| 2 | **缺乏历史验证** | 用户无法知道"如果相信这个预测系统去操作，历史表现如何"。无法量化策略的夏普比率、最大回撤 |
| 3 | **预测与实践脱节** | 系统每天生成 bullish/bearish 预测和置信度，但用户不知道这些信号"值多少钱"——在真实市场中按预测操作能否赚钱 |
| 4 | **无实盘辅助** | 用户看到预测后仍需手动决策买卖，缺乏系统化的交易建议（买什么、买多少、何时入场） |
| 5 | **一视同仁的权重** | 趋势市场与震荡市场使用同一套权重，但在不同市场状态下有效的信号完全不同 |

### 1.3 解决方向

```
痛点1+5 → 回测引擎提供冷启动权重 + 市场状态自适应
痛点2+3 → 回测引擎产生历史资金曲线、夏普比率等量化指标
痛点4   → 纸面交易每日生成买卖建议，虚拟账户模拟运行
```

## 2. 核心目标

### 2.1 产品目标

| 目标 | 衡量标准 |
|------|---------|
| **策略可信度可量化** | 每只自选股展示历史回测的夏普比率、最大回撤、胜率、年化收益 |
| **预测信号可执行化** | 每天自动将 bullish/bearish 预测转化为具体的"买X股@Y元"或"卖X股"建议 |
| **学习冷启动消除** | 新自选股直接使用回测最优权重，而非从 1.0 随机摸索 |
| **市场状态自适应** | 趋势/震荡/高波动自动切换不同权重矩阵，提升不同行情下的准确率 |

### 2.2 用户目标

| 场景 | 用户期望 |
|------|---------|
| 盘前查看建议 | 早上打开系统，看到今日每只股票的买卖建议（方向+数量+价格） |
| 验证系统可靠性 | 在真正投钱之前，先看到"假设按预测操作，历史能赚多少" |
| 辅助买卖决策 | 做多/做空犹豫时，参考系统给出的建议和置信度 |
| 跟踪虚拟表现 | 看到虚拟账户的资金曲线，与真实持仓形成对比 |
| 定期优化参数 | 每周/每月运行回测，让系统持续学习最优策略 |

## 3. 用户场景

### 场景A: 盘后自动执行纸面交易

```
角色: 系统 (自动)
时间: 每个交易日收盘后，sync_all.py 运行完成后
前置条件: 日K线已更新，10天预测已生成

1. sync_all.py 8步流程完成
2. 系统自动调用 paper_trading.py 生成当日建议
3. 系统根据建议自动执行模拟交易:
   - 兴业银行: 预测 bullish, 置信度65%
     → 自动买入300股 @ ¥17.35, 仓位12%
   - 农业银行: 预测 bearish, 置信度58%, 已有持仓500股
     → 自动卖出500股 @ ¥3.72
   - 招商银行: 预测 neutral, 置信度42%
     → 不操作
4. 更新虚拟账户: 扣减/增加现金 → 更新持仓 → 记录交易 → 生成快照
5. 用户次日打开系统，在"模拟交易"页看到已自动执行的交易结果
```

### 场景B: 用户查看自动交易结果

```
角色: 投资用户
时间: 次日早上查看
前置条件: 前一日纸面交易已自动执行

1. 用户打开系统，点击"模拟交易 → 纸面交易"
2. 看到昨日自动执行的交易摘要:
   - ✅ 买入 兴业银行 300股 @ ¥17.35
   - ✅ 卖出 农业银行 500股 @ ¥3.72
   - — 招商银行 持有观望
3. 用户查看更新后的虚拟账户:
   - 当前总资产: ¥108,500 (+8.5%)
   - 虚拟持仓: 兴业银行300股、建设银行1000股
4. 用户参考自动交易结果，辅助真实账户决策

### 场景C: 首次运行回测验证

```
角色: 投资用户
时间: 任意时间，首次使用回测功能
前置条件: K线数据至少252天

1. 用户点击导航栏"模拟交易 → 回测分析"
2. 界面显示: "未运行过回测，点击开始"
3. 用户选择参数: 训练窗口252天 / 测试窗口21天 → 点击"运行回测"
4. 状态指示器变为"●运行中"，显示进度 "3/8 兴业银行..."
5. 约90秒后完成，显示结果:
   - 夏普比率: 0.85 | 最大回撤: -12.5% | 胜率: 58.3% | 年化: +15.7%
   - 兴业银行: 原始预测准确率55.2% → 回测优化后62.1% (↑6.9%)
   - 市场状态权重: 趋势市场MACD权重1.52，震荡市场RSI权重1.45...
6. 用户看到 improves → 系统自动将回测权重写入 learning_params
7. 下次 sync_all.py 运行时，MWU 从优化权重冷启动
```

### 场景C: 跟踪虚拟资金曲线

```
角色: 投资用户
时间: 日常查看
前置条件: 纸面交易已执行多笔

1. 用户点击"模拟交易 → 交易历史"
2. 看到资金曲线图:
   - 蓝色实线: 纸面账户 ¥100,000 → ¥108,500 (+8.5%)
   - 灰色虚线: 买入持有基准 ¥100,000 → ¥103,200 (+3.2%)
3. 统计摘要: 45笔交易 / 胜率58.3% / 盈亏比1.32
4. 用户可以切换时间范围: 30天 / 90天 / 180天
5. 用户对比真实持仓 (本页可对比) → 判断预测系统是否跑赢买入持有
```

## 4. 功能点清单

### 4.1 回测引擎

| # | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F1 | Walk-forward 滚动优化 | P0 | 252天训练→21天测试→前滚21天，遍历全部历史 |
| F2 | 10信号网格搜索权重 | P0 | 权重范围0.1-1.5，随机采样+贝叶斯优化，限2000次评估/股 |
| F3 | 市场状态检测 | P1 | 基于ADX和波动率自动标注 trend/ranging/volatile |
| F4 | 状态专属权重学习 | P1 | 每状态独立训练一套权重矩阵 |
| F5 | 性能指标计算 | P0 | 夏普比率、最大回撤、胜率、盈亏比、Calmar、年化收益 |
| F6 | 权重对比分析 | P1 | 原始MWU权重 vs 回测优化权重，计算改善幅度 |
| F7 | 结果持久化 | P0 | 写入 learning_params.backtest_weights / regime_weights |
| F8 | 异步运行 | P0 | 通过 subprocess 调用，不阻塞 API 主线程 |
| F9 | 进度查询 | P1 | 实时查询运行状态 (running/progress/completed/error) |
| F10 | 历史记录 | P2 | 每次回测运行记录可追溯 |

### 4.2 纸面交易

> **核心原则**：纸面交易完全由算法自动执行，**不需要用户确认**。每日从 `daily_predictions` 表中读取当天预测（该数据源来自 `gen_multi_day_pred()` 生成的10天滚动预测），自动决策并执行模拟买卖。用户仅查看结果，作为真实投资的参考。

> **数据一致性约束（🔴 MUST）**：纸面交易使用的预测数据**必须**与前端智能预测页展示的10天预测数据**来自同一数据源**——即 `daily_predictions` 表中 `date = TODAY` 的记录。纸面交易的买卖决策与预测展示的 direction / confidence / entry_zone 保持一致，不可使用独立计算或缓存副本。

| # | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| F11 | 虚拟账户管理 | P0 | 初始化 ¥100,000，支持重置 |
| F12 | 每日建议自动生成 | P0 | 从 `daily_predictions` 读取当日预测（direction/confidence/entry_zone），与前端10天预测数据同源 |
| F13 | 凯利仓位计算 | P0 | 基于预测 `confidence` 计算仓位：`confidence × 2 - 1`，裁剪到0~0.3，整手取整 |
| F14 | 算法自动执行交易 | P0 | 以预测 `entry_zone` 为执行价，sync_all.py 完成后自动调用 |
| F15 | 手续费模拟 | P0 | 佣金0.03% + 卖出印花税0.1%，与真实费率一致 |
| F16 | 虚拟持仓管理 | P0 | 实时市值/浮盈亏计算 |
| F17 | 资金曲线图 | P1 | 纸面账户 vs 买入持有基准，支持30/90/180天 |
| F18 | 表现指标 | P1 | 夏普、最大回撤、胜率、盈亏比、累计收益 |
| F19 | 交易历史 | P1 | 全部交易记录，支持按代码筛选 |
| F20 | 手动重置 | P2 | 支持重置虚拟账户（保留交易历史） |

## 6. 约束限制

### 6.1 性能约束

| 约束项 | 值 | 原因 |
|--------|-----|------|
| 回测单次运行时间 | < 300秒 (API超时) | 避免HTTP连接超时 |
| 回测每股票权重评估 | ≤ 2000次 | 网格搜索 + 随机采样降维 |
| 训练窗口最小K线 | ≥ 126天 | 信号计算至少需要14天，默认252天需要≥1年数据 |
| 训练窗口最大可用K线 | ≤ 2000天 | 当前实际最大约2000条（约8年历史），超出部分不可用 |
| 纸面交易建议响应 | < 2秒 | 页面加载体验 |
| 资金曲线数据点 | ≤ 365（前端懒加载90天） | 避免Chart.js渲染瓶颈 |

### 6.2 业务约束

| 约束项 | 规则 |
|--------|------|
| 虚拟账户初始资金 | 固定 ¥100,000 |
| 买入最小单位 | 100股（整手） |
| 单只股票最大仓位 | 总资产的30%（凯利上限） |
| 模拟交易时间 | 仅交易日，不模拟盘后 |
| 建议生成时机 | 每日一次，基于最新预测 |
| 手续费费率 | 复用 `config.json` 中的 `fee_rates` |

### 6.3 安全约束

| 约束项 | 规则 |
|--------|------|
| 数据隔离 | 纸面交易使用独立表（paper_前缀），不修改真实持仓数据 |
| 重置保护 | 重置账户需二次确认 |
| 交易执行保护 | 资金不足、持仓不足时拒绝执行并提示 |
| 回测并发 | 同一时间只允许一个回测任务运行 |

## 7. 与现有系统的边界

### 7.1 数据流边界

```
[已有] sync_all.py Step 1-4: 获取K线 + 回填验证
        │
[新增] sync_all.py Step 5+: 读取 backtest_weights 冷启动
        │
[已有] sync_all.py Step 6: 生成10天预测 → daily_predictions
        │
[新增] paper_trading.py: 读取预测 → 生成建议 → paper_suggestions
        │
[新增] server_v2.py: /api/v2/paper/suggestions → 前端纸面交易页
        │
[已有] server_v2.py: /api/trigger/predict → 已有功能，不修改
```

### 7.2 不修改的部分

| 模块 | 说明 |
|------|------|
| `sync_all.py` Step 1-4, Step 6-8 | 核心预测流程不变 |
| `server_v2.py` 现有30+端点 | 行为完全不变 |
| `db_helper.py` 现有17张表的CRUD | 不修改，只新增函数 |
| `daily_predictions` 表 | 预测生成逻辑不变 |
| `positions/trades` 表 | 真实持仓完全不受纸面交易影响 |
| 广发对账单流程 | 不受影响 |

## 9. 成功指标

| 指标 | 目标值 | 衡量方法 |
|------|--------|---------|
| 回测优化提升 | 改善幅度 > 2% (方向准确率) | 对比原始MWU vs 回测优化后的历史验证 |
| 纸面交易胜率 | > 50% | 虚拟账户中盈利交易占比 |
| 回测运行时间 | < 120秒 (8只股票) | 计时 |
| 用户采纳率 | 用户每天查看纸面交易建议 | 日志分析（建议生成 → 建议执行转化率） |

## 10. 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 网格搜索计算量过大 | 中 | 回测超时 | 限2000次评估/股，numpy向量化 |
| K线数据不足252天 | 中 | 无法回测 | 跳过该股票，提示用户"数据不足" |
| 纸面交易与真实持仓混淆 | 低 | 决策错误 | UI用橙色标识区分，表用paper_前缀 |
| 回测过拟合 | 中 | 实盘表现差于回测 | Walk-forward滚动验证 + 等权基线对比 |
| 用户过度依赖模拟建议 | 中 | 投资亏损 | 免责声明 + "仅供参考"提示 |

## 11. 术语表

| 术语 | 定义 |
|------|------|
| **Walk-forward** | 滚动窗口优化：训练期→测试期→前滚→重复，防止过拟合 |
| **冷启动** | 系统首次为新股票生成预测，尚无历史学习经验 |
| **纸面交易 (Paper Trading)** | 用虚拟资金模拟真实交易，不涉及真实金钱 |
| **凯利公式** | 根据胜率和赔率计算最优下注比例的公式 |
| **夏普比率** | 收益 / 波动率，衡量风险调整后收益 |
| **事件溯源** | 每次交易为一条不可变事件，通过重放事件计算当前状态 |
