# 附录 F — 已知问题与限制

> **关联规范**: 全站引用 | **状态**: 持续更新

---

## 1. 数据质量问题

| 问题 | 严重度 | 现状 | 影响范围 |
|------|--------|------|----------|
| PE/PB/股息率部分为0 | 🔴 高 | 部分数据源不提供实时PE/PB/DY | StockData页面显示"0" |
| 行情非实时 | 🟡 中 | 仅通过定时刷新获取，非推送 | 所有行情展示延迟5-30分钟 |
| K线前复权一致性 | 🟡 中 | 多数据源复权算法不同 | K线图不同源数据有偏差 |
| 新闻情感准确率 | 🟡 中 | 基于关键词规则，非NLP模型 | 部分情感标注不准确 |

## 2. 架构问题

| 问题 | 严重度 | 现状 | 影响范围 |
|------|--------|------|----------|
| sync_all模块级执行 | 🟡 中 | import即开始执行 | 启动时意外触发同步 |
| 全局锁限制并发 | 🟡 中 | 长耗时任务互斥 | 无法同时回测和刷新 |
| SQL f-string注入风险 | 🟡 中 | db_helper使用f-string拼SQL | 理论上存在注入风险 |
| 子进程超时硬编码 | 🟡 中 | timeout参数硬编码 | 大数据量同步可能超时 |
| 无结构化日志系统 | 🟢 低 | print输出到控制台 | 问题排查困难 |
| 双写不一致 | 🟡 中 | JSON和SQLite同时写入 | data/system_data.json可能不同步 |

## 3. 性能限制

| 指标 | 限制值 | 说明 |
|------|--------|------|
| 自选股数量 | ≤20只 | 过多会导致K线采集时间过长 |
| 回测最大窗口 | 504天 | 超过可能导致内存溢出 |
| 纸面交易建议数 | ≤50条/次 | 避免执行延迟 |
| 新闻缓存 | 最近30天 | 只保存一个月内的新闻 |
| K线数据 | 最近5年 | 超过5年的数据可能不完整 |

## 4. 兼容性限制

| 限制 | 说明 |
|------|------|
| 仅支持广发证券对账单 | update_from_statement.py专为广发格式编写 |
| 仅支持Chrome/Firefox/Edge | 前端使用现代CSS特性 |
| 仅支持Windows | scheduler.py依赖Windows任务计划程序 |
| Python ≥ 3.10 | 使用match语法和新特性 |
| Node.js ≥ 18 (构建环境) | Vite 5需要 |
