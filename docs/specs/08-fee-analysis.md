## 1. 业务需求说明书

### 1.1 业务背景

交易产生的各项费用（佣金、印花税、过户费等）对投资收益有显著影响，需要结构化分析费用的组成和趋势。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 费用汇总 | 总佣金、总印花税、其他费用、费用合计 |
| 费用构成 | 环形图展示各费用类别占比 |
| 逐月趋势 | 堆叠柱状图展示费用月度变化 |
| 个股明细 | 各股票的费用明细和费率估算 |

## 3. 业务逻辑

### 3.1 费用统计计算

```javascript
// 遍历所有持仓（当前持仓 + 已清仓）
// 汇总 commission, stamp_tax, other_fees
// 费率估算 = 费用合计 / 交易总额
export function calcFees(positions, closedPositions) {
    let totalCommission = 0, totalStamp = 0, totalOther = 0
    for (const p of [...positions, ...closedPositions]) {
        totalCommission += p.total_commission || 0
        totalStamp += p.total_stamp_tax || 0
        totalOther += p.total_other_fees || 0
    }
    return { totalCommission, totalStamp, totalOther, total: totalCommission + totalStamp + totalOther }
}
```

## 4. 费率配置

系统费率配置在 `data/config.json` 中：

```json
{
  "fee_rates": {
    "transfer_fee_per_1000": 1.0,   // 过户费 每千股1元
    "regulatory_fee_rate": 0.00002,  // 证管费 0.002%
    "handling_fee_rate": 0.0000487   // 经手费 0.00487%
  }
}
```

## 5. 交互流程

```
路由 /fees → 组件挂载
  → useDataStore 获取所有持仓和交易数据
  → 计算汇总统计 → 4个统计卡片
  → 按股票分组计算费用 → 费用明细表
  → Chart.js onMounted 绘制环形图 + 堆叠柱状图
```
